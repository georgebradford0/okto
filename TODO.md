# TODO

- [ ] Front the push-notification relay (`octorelay.directto.link`) with Cloudflare to absorb DDoS / bad-signature floods before they reach the t4g.nano's CPU credits. Today the box is bare on a public Elastic IP; if `/notify` ever attracts spammers, Ed25519 verifies are cheap but can still chew through burst credits at high enough volume. Cloudflare in front gives free L7 caching, rate-limits, bot challenges, and a single knob to drop traffic without SSH'ing the instance.
- [ ] Lair → child liveness probe. Today lair's 10s poller only asks the local Docker daemon whether each `octo.managed=1` container exists / is running — it never talks to the agent process itself, and remote agents are skipped entirely (registry status is frozen at `register_remote_agent` time). Add a per-child reachability check during each poll: HTTP GET each child's `/health` (already served at `lair/src/agent.rs:89`) or do a Noise handshake against its published host:port. Surface failures in `ContainerInfo.status` so mobile shows a child that's wedged / unreachable distinctly from one Docker says is stopped. Especially important for remote agents where lair currently has zero visibility into VM health.
- [ ] Replace `GH_TOKEN` PAT-in-env with a GitHub App + installation tokens. Today operators set `GH_TOKEN=ghp_…` via `octo init --env` and it's forwarded verbatim to every child — a long-lived classic PAT with `repo`-wide scope that lands in `docker inspect` on lair and every child, with no per-agent or per-repo scoping and no expiry. Replace with the App-installation-token flow GitHub Actions uses internally: (a) one-time setup via the GitHub App manifest flow (`octo gh-app setup` opens a URL, operator clicks Install on github.com, picks repos, redirects back; CLI captures App ID + private-key PEM into `config.json`); (b) lair becomes a token broker — signs a short RS256 JWT with the App private key (~10 min), exchanges it at `POST /app/installations/{id}/access_tokens` for a ~1-hour installation token scoped to the App's selected repos; (c) `exec_create_agent` injects a freshly-minted installation token instead of a PAT, and lair exposes a `request_gh_token` RPC over the existing Noise tunnel so long-running children can refresh on demand. Wins: short-lived tokens (blast radius ~1h instead of "until manually revoked"), per-repo scoping at install time, no manual PAT pasting, no `--env GH_TOKEN=…` opt-in. Implementation: ~200 lines of Rust + one `jsonwebtoken` dep + a CLI subcommand for the manifest flow. As an interim cheap stopgap consider rewriting HTTPS git URLs to SSH form inside `exec_create_agent` and mounting lair's existing `/data/ssh_id_ed25519` into children (covers the clone path with zero new auth — but doesn't fix `gh` CLI calls, so it's a partial answer at best).
